## 1. 디자인 철학 & 무드

### 1.1 핵심 컨셉

**"따뜻한 햇살 아래 펼쳐지는 날씨 이야기"**

날씨 에너지 앱은 아이들이 경제를 배우는 공간이지만, 동시에 **편안하게 쉬어갈 수 있는 힐링 공간**이어야 합니다. 마치 동화책을 펼쳐놓고 이야기 속으로 들어가듯, 자연스럽고 포근한 경험을 제공합니다.

### 1.2 비주얼 키워드

| 키워드 | 설명 | 디자인 적용 |
|--------|------|-------------|
| **따뜻함** | 차갑지 않고 온기 있는 | 따뜻한 색온도, 부드러운 그림자 |
| **편안함** | 긴장되지 않고 자연스러운 | 둥근 모서리, 넉넉한 여백 |
| **밝음** | 희망적이고 명랑한 | 선명한 색상, 높은 채도 |
| **친근함** | 가깝고 다가가기 쉬운 | 친근한 일러스트, 쉬운 말 |

### 1.3 무드보드

**자연 속 하루의 흐름**

```
[아침 - 부드러운 오렌지빛 하늘]
☀️ 해가 떠오르는 순간
밝고 희망찬 에너지

[낮 - 맑고 푸른 하늘과 구름]
💨☁️ 시원한 바람이 부는 날
활기차고 경쾌한 분위기

[저녁 - 따뜻한 석양과 구름]
🌤️ 노을이 지는 풍경
포근하고 평화로운 느낌

[밤 - 별이 반짝이는 하늘]
✨ 고요하고 차분한 밤
안정적이고 편안한 시간
```

**동화책 속 날씨 풍경**
- 둥글둥글한 구름들이 떠다니는 하늘
- 부드러운 언덕과 나무들
- 귀여운 날씨 친구들 (태양, 비, 바람, 온도)
- 계절마다 변하는 자연의 모습

### 1.4 디자인 원칙

#### ✅ 해야 할 것

1. **둥글게**: 모든 요소는 둥근 모서리를 가져요
2. **밝게**: 색상은 선명하고 활기차게
3. **넓게**: 여백을 충분히 주어 숨을 쉴 수 있게
4. **쉽게**: 직관적으로 이해할 수 있게
5. **따뜻하게**: 차갑지 않고 포근한 느낌으로

#### ❌ 하지 말아야 할 것

1. **각지게**: 날카로운 모서리와 뾰족한 형태
2. **어둡게**: 무겁고 우울한 색상
3. **빽빽하게**: 답답하고 복잡한 레이아웃
4. **어렵게**: 복잡한 용어와 구조
5. **차갑게**: 기계적이고 냉정한 느낌

### 2.5 색상 사용 가이드

#### 색상 조합 규칙

**✅ 좋은 조합**
```
1. 배경 + 메인 색상
   #FFFCF7 (배경) + #FFB547 (태양)
   
2. 카드 + 상태 색상
   #FFFFFF (카드) + #FFE9E9 (상승 배경)
   
3. 텍스트 + 배경
   #2C3E50 (텍스트) + #FFFCF7 (배경)
```

**❌ 피해야 할 조합**
```
1. 비슷한 밝기의 색상
   #FFB547 + #FFC078 (구분이 안됨)
   
2. 너무 강한 대비
   #FF0000 + #FFFFFF (눈이 아픔)
   
3. 어두운 색상 과다 사용
   여러 어두운 색을 함께 쓰지 않기
```

#### 색상 접근성

**텍스트 대비율**
```
큰 텍스트 (18pt 이상): 3:1 이상
작은 텍스트 (18pt 미만): 4.5:1 이상
중요 정보: 7:1 권장

예시:
✅ #2C3E50 (어두운 회색) on #FFFCF7 (배경) = 10.5:1
✅ #495057 (회색) on #FFFFFF (카드) = 8.2:1
❌ #ADB5BD (연한 회색) on #FFFFFF = 2.7:1 (본문에 사용 금지)
```

## 3. 타이포그래피

### 3.1 폰트 패밀리

#### 메인 폰트

**본문: Pretendard**
```
특징:
- 한글/영문/숫자 모두 자연스러움
- 다양한 굵기 지원 (100~900)
- 가독성이 뛰어남
- 아이들이 읽기 편한 자간

다운로드: https://github.com/orioncactus/pretendard
라이선스: SIL Open Font License
```

**숫자 강조: Pretendard (Tabular Numbers)**
```
사용처: 가격, 금액, 퍼센트
특징: 숫자 폭이 일정해서 자릿수 변경시에도 레이아웃 안정
예: 1,200 → 1,250 (자리가 흔들리지 않음)
```

#### 대체 폰트

**시스템 폰트 스택**
```
font-family: 
  "Pretendard", 
  -apple-system, 
  BlinkMacSystemFont,
  "Apple SD Gothic Neo",
  "Malgun Gothic", 
  sans-serif;
```

## 4. 일러스트레이션 스타일

### 4.1 일러스트 원칙

**"플랫하고 밝은 동화책 그림"**

#### 스타일 특징

1. **평면적 (Flat)**: 그라데이션 최소, 선명한 색 블록
2. **둥근 형태**: 모든 모서리는 라운드 처리
3. **단순한 디테일**: 복잡하지 않고 깔끔하게
4. **밝은 색상**: 채도 높은 명랑한 색
5. **친근한 표정**: 부드럽고 다정한 느낌

### 4.6 일러스트 제작 가이드

#### ✅ 해야 할 것

1. **단순하게**: 디테일을 최소화
2. **밝게**: 채도 높은 색상 사용
3. **둥글게**: 모든 모서리 라운드 처리
4. **균형있게**: 좌우 대칭 또는 시각적 균형
5. **친근하게**: 귀엽고 다정한 느낌

#### ❌ 하지 말아야 할 것

1. **복잡하게**: 너무 많은 디테일
2. **어둡게**: 채도 낮거나 어두운 색
3. **각지게**: 날카로운 모서리
4. **사실적으로**: 리얼리즘 지양
5. **무섭게**: 위협적이거나 불안한 느낌

```

□ 모든 모서리가 둥근가?
□ 배경과 잘 어울리는가?
□ 아이들이 이해하기 쉬운가?
□ 따뜻하고 친근한 느낌인가?
```

## 6. 애니메이션 & 모션

### 6.1 모션 원칙

**"자연스럽고 부드러운 움직임"**

#### 모션 철학

1. **자연스러움**: 실제 물리 법칙을 따르는 듯한 움직임
2. **부드러움**: 갑작스럽지 않고 매끄러운 전환
3. **의미있음**: 모든 애니메이션은 목적이 있어야 함
4. **적절함**: 과하지 않고 필요한 만큼만
5. **즐거움**: 작은 즐거움을 주는 디테일

### 6.10 애니메이션 가이드라인

#### ✅ 해야 할 것

1. **목적 있게**: 모든 애니메이션은 이유가 있어야 함
2. **부드럽게**: 자연스러운 easing 사용
3. **빠르게**: 대부분 300ms 이내로
4. **피드백**: 사용자 액션에 즉각 반응
5. **일관성**: 같은 동작은 같은 애니메이션

#### ❌ 하지 말아야 할 것

1. **과하게**: 너무 많은 애니메이션
2. **느리게**: 600ms 이상은 지루함
3. **선형으로**: linear easing 지양 (특수한 경우 제외)
4. **불필요하게**: 의미 없는 장식용 애니메이션
5. **방해하게**: 콘텐츠 읽기를 방해하는 움직임

#### 성능 고려사항

```
✅ 하드웨어 가속 사용
   - transform (translate, scale, rotate)
   - opacity
   
❌ 피해야 할 속성
   - width, height
   - margin, padding
   - top, left, right, bottom
   
최적화:
   - will-change 적절히 사용
   - 60fps 유지
   - 배터리 소모 최소화
```
